▶ Radius Datalake 数据湖平台
核心开发 | 2024 - 至今 | 生产系统

技术栈：Airflow + Iceberg + Trino + PyIceberg + Polars + PyArrow + RustFS(S3) + Python

  • 链路闭环：设计并落地 10 DAG 主链路（4 个 core + 6 个 business），当前仓库总 dag_id 为 13（含日志链路）。
  • 配置驱动：从业务 API 拉取数据源与字典配置，JWT Token 自动刷新并缓存到 Airflow Variables，降低手工维护成本。
  • 入湖引擎：MySQL 流式游标批量入湖（默认 10k/批，max_active_tasks=10）+ Mongo 原始文档入湖（raw_document）+ Iceberg schema evolution。
  • 规则引擎：字典抽取 SQL 由规则驱动自动生成，MySQL 多表关联采用 BFS 自动找路径，失败场景自动回退 UNION ALL，显著降低人工拼 SQL 成本。
  • 回传容错：按分类并行、批次重试、错误隔离（默认 100 条/批）；核心 DAG 默认手动触发（schedule=None），可按部署策略切换为定时调度。
